## Git Payload

Migrate source, tests, protocol and architecture docs, Docker definition, plugin packaging metadata, and portable documentation.

Do not migrate through Git:

- Operator credentials, tokens, recovery keys, or Windows credential-store entries
- Active VibeBus task, inbox, ACK, reservation, delivery, subscription, or runtime SQLite state
- Local `.vibebus-local`, backup directories, databases, build output, or Codex session/plugin cache

The tracked path `tests/credential_vault.rs` is a credential-vault test file by name. It must not be treated as a live secret without content review, and no secret value should be printed during review.

- Primary local runtime: VibeBus CLI/MCP from source

- Runtime root and credential state must be recreated on Mac, not copied from Windows

## Restore Commands On Mac

```bash
git clone VIBEBUS_PRIVATE_URL ~/Projects/VibeBus
cd ~/Projects/VibeBus
git fetch --all --tags --prune
git switch migration/pc-to-mac-20260719
git rev-parse HEAD
```

If global Cargo is unavailable on Windows, the PC checkout previously used repository-local Cargo tooling. On Mac, install a normal Rust toolchain unless the project later documents a local toolchain bootstrap.

- Start CLI/MCP on the Mac project root.
- Re-register Mac-local identity and credentials.
- Confirm credential display remains redacted.

- Treat Windows PowerShell acceptance scripts as Windows-only until PowerShell Core and macOS compatibility is verified.

## Known Blockers

- No migration remote push has been performed in this PC phase.
- Confirm repository visibility before pushing.
- Mac runtime identity, credentials, and active collaboration state must be recreated, not copied.

## Rollback

- Keep this PC checkout unchanged for at least two weeks after Mac acceptance.
- Use the migration branch, annotated tag, and local Git bundle as source-code restore points.
- If Mac restore fails, delete only the new Mac checkout and retry from Git.
